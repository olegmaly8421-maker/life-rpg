# ⬡ Life RPG 2.0

RPG-гейміфікація життя з хмарою, синхронізацією між пристроями, входом і повністю редагованою структурою.

## Що вміє

- ☁️ **Хмара (Supabase)** — дані синхронізуються між усіма пристроями
- 🔐 **Вхід через email** — тільки ти бачиш свої дані
- ⚙️ **Редактор** — додавай/зміюй характеристики, підпункти, дії, квести прямо в застосунку
- 📱 **PWA** — встановлюється на телефон/планшет як застосунок
- 📊 Dashboard, детальні стати, щоденний лог, графіки прогресу, квести

---

## ЗАПУСК ЛОКАЛЬНО (на комп'ютері)

### 1. Встанови залежності
```
npm install
```

### 2. Перевір файл `.env`
Він уже містить твої дані Supabase. Якщо його немає — скопіюй:
```
copy .env.example .env
```
і встав туди свій URL та ключ.

### 3. Запусти
```
npm run dev
```

### 4. Відкрий
```
http://localhost:5173
```

Перший раз — натисни **«Немає акаунту? Створити»**, введи email і пароль (мін. 6 символів), зареєструйся, потім увійди. Персонаж створиться автоматично зі стартовими характеристиками.

## РОЗМІСТИТИ В ІНТЕРНЕТІ (Vercel) — щоб працювало з телефону завжди

### 1. Залий код на GitHub
Створи репозиторій і завантаж туди цю папку (без `node_modules` і `.env`).

### 2. Підключи Vercel
- Зайди на https://vercel.com, увійди через GitHub
- **Add New → Project** → обери свій репозиторій
- Framework: **Vite** (визначиться сам)

### 3. Додай змінні середовища
У налаштуваннях проєкту Vercel → **Environment Variables** додай два ключі:

```
VITE_SUPABASE_URL = https://ynesobdtewvzcsadxumo.supabase.co
VITE_SUPABASE_ANON_KEY = (твій anon ключ eyJ...)
```

### 4. Deploy
Натисни **Deploy**. Через хвилину отримаєш посилання типу `life-rpg.vercel.app`.

## ВСТАНОВИТИ НА ТЕЛЕФОН (PWA)

1. Відкрий своє Vercel-посилання в браузері телефону (Chrome/Safari)
2. **Android:** меню (⋮) → «Додати на головний екран»
3. **iPhone:** кнопка «Поділитись» → «На екран Домівки»

З'явиться іконка як у звичайного застосунку. Дані ті самі, що на комп'ютері — синхронізуються через хмару.

## Безпека

Після налаштування зроби ротацію anon-ключа в Supabase (Settings → API → Reset), бо ключ був у листуванні. Онови його в `.env` і у Vercel.
